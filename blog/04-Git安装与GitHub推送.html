Git 安装配置与 GitHub 推送完全指南

2026-06-02 | Git GitHub 环境变量 教程

一、Git 安装选择

方案 A：安装版（推荐新手）

  • 官网下载：https://git-scm.com/download/win
  • 安装路径：D:\Apps\ProgramFiles\Git\
  • 关键步骤：选择 *"Git from the command line and also from 3rd-party software"*（自动加入 PATH）

方案 B：Portable 版（绿色版）

  • 下载 PortableGit-*-64-bit.7z.exe
  • 解压到：D:\Apps\Portable\Git\
  • 必须手动添加环境变量

二、环境变量配置（Portable 版必做）

判断终端类型

终端提示符特征可用命令
CMDC:\Users\HP>setx
PowerShellPS C:\Users\HP>[Environment]::SetEnvironmentVariable

CMD 添加 PATH（推荐）

setx Path "%Path%;D:\Apps\Portable\Git\bin"

看到 "成功: 指定的值已得到保存。" 后，关闭终端重新打开。

验证安装

git --version

输出 git version 2.49.0.windows.1 即成功。

三、Git 身份配置（仅需一次）

git config --global user.name "你的GitHub用户名"
git config --global user.email "你的邮箱@example.com"

四、从 GitHub 克隆项目到本地

cd D:\Projects\ByLifeCycle\03-Personal\Portfolio
git clone https://github.com/用户名/仓库名.git
本地留存是必需的：GitHub 是展示平台，不是唯一备份。

五、修改 → 推送标准工作流

git status
git add .
git commit -m "更新技能图谱：添加Python项目"
git push

首次推送需授权

执行 git push 后若提示浏览器授权，点击 "Sign in with your browser" 即可。

六、常用命令速查

命令作用
git status查看修改状态
git add .添加所有修改
git commit -m "说明"提交到本地
git push推送到远程
git pull拉取远程最新
git log查看提交历史
git clone 克隆远程仓库
