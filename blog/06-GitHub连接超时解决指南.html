GitHub 连接超时解决指南：5 种方法从快到稳

📅 2026-06-02 ✍️ binqi ⏱️ 阅读约 5 分钟

推送代码时遇到 Request: Could not find any IP that can be successfully connected 或 HTTP connection timed out，这是国内访问 GitHub 的经典问题。本文总结 5 种解决方法，从最快到最稳定。

一、方法总览

方法 适用场景 复杂度 效果
重启加速器 开了梯子但突然断连 ⭐ 最简单 立竿见影
换网络（手机热点） 当前网络被限制 ⭐ 简单 临时解决
检查/修正代理端口 代理配置错误 ⭐⭐ 中等 永久解决
取消代理直连 代理反而导致问题 ⭐ 简单 测试用
改用 SSH 推送 长期稳定使用 ⭐⭐⭐ 较复杂 最稳定

二、方法一：重启加速器（首推，最快）

💡 问题原因：加速器/梯子运行久了，连接池耗尽或节点失效，导致 GitHub IP 连不上。

操作步骤

  1. 完全退出加速器（右键托盘图标 → 退出，不是最小化）
  2. 等待 5 秒，确保进程彻底结束
  3. 重新打开加速器，手动切换一个节点（建议香港/日本/新加坡）
  4. 浏览器打开 github.com 确认能访问
  5. 回到终端重新执行 git push
⚠️ 注意：很多加速器的"自动选择节点"功能可能选到刚好挂掉的节点，手动切换更可靠。

三、方法二：换网络环境

如果重启加速器无效，尝试：

  • 手机开热点给电脑
  • 切换公司/学校/家里的不同 WiFi
  • 关闭 WiFi 用有线连接

某些运营商（如移动宽带）对 GitHub 限制更严，电信/联通通常好一些。

四、方法三：检查 Git 代理配置

查看当前设置

git config --global http.proxy
git config --global https.proxy

常见加速器端口对照

加速器HTTP 代理端口设置命令
Clash7890git config --global http.proxy http://127.0.0.1:7890
Clash Verge7897git config --global http.proxy http://127.0.0.1:7897
V2RayN10809git config --global http.proxy http://127.0.0.1:10809
Shadowsocks1080git config --global http.proxy socks5://127.0.0.1:1080

同时设置 HTTPS

git config --global https.proxy http://127.0.0.1:7890

五、方法四：取消代理（代理反而捣乱时）

如果加速器没开，或者之前设了错误的代理，直接取消：

git config --global --unset http.proxy
git config --global --unset https.proxy

取消后尝试直连，如果还是超时，说明需要开加速器。

六、方法五：改用 SSH 推送（长期最佳）

SSH 比 HTTPS 更稳定，且不需要每次输入密码或浏览器授权。

步骤 1：生成 SSH 密钥

ssh-keygen -t ed25519 -C "你的邮箱@example.com"
# 一路回车，使用默认路径

步骤 2：复制公钥到 GitHub

cat C:\Users\你的用户名\.ssh\id_ed25519.pub

把输出的内容完整复制，粘贴到 github.com/settings/keys → New SSH key

步骤 3：测试连接

ssh -T git@github.com
# 看到 "Hi binqi! You've successfully authenticated" 即成功

步骤 4：修改仓库地址为 SSH

git remote set-url origin git@github.com:binqi/ziyuanzhan.github.io.git

步骤 5：以后直接 push

git push
# 无需浏览器授权，一次配置永久生效

七、快速排查流程

git push 报错超时？
    │
    ├─ 开了加速器？→ 重启加速器，换节点，再试
    │
    ├─ 没开加速器？→ 手机开热点，再试
    │
    ├─ 还是不行？→ 检查 git proxy 设置
    │               ├─ 有设置 → 确认端口和加速器一致
    │               └─ 没设置 → 考虑配 SSH 长期解决
    │
    └─ 浏览器能开 github.com？→ 能 = 网络正常，检查 Git 配置
                               → 不能 = 网络问题，换热点或加速器

八、总结建议

  • 应急：重启加速器 → 换热点 → 检查代理端口
  • 长期：配置 SSH 密钥，告别 HTTPS 超时烦恼
  • 备用：手机热点是终极兜底方案
📌 我的选择：日常用加速器 + HTTPS，遇到超时先重启加速器；同时配置了 SSH 作为备用，双保险。
