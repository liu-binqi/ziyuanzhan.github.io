一个域名能做什么？从 ziyuanzhan.top 说起

📅 2026-06-01 ✍️ binqi ⏱️ 阅读约 10 分钟

我花了几十块钱买了 ziyuanzhan.top 这个域名，刚开始只是想做个简历页。研究了一圈后发现，<一个域名的潜力远超想象。这篇文章记录我的学习过程，也希望能帮到你。

一、域名是什么？

简单来说，域名就是网站的门牌号。用户输入 ziyuanzhan.top，DNS 系统就会告诉浏览器："这个域名对应的服务器在这，去那找网页"。

类比理解：
域名 = 小区名（如"阳光花园"）<
IP 地址 = 精确的门牌号（如"北京市海淀区 XX 路 123 号"）<
DNS = 快递查询系统（把小区名翻译成门牌号）

二、一个域名 = 无限个子域名

很多人以为一个域名只能做一个网站，其实一个主域名可以分出无数个"子域名"，每个子域名指向不同的服务。

以我的 ziyuanzhan.top 为例：

子域名 用途 状态
ziyuanzhan.top 个人主页 / 简历 ✅ 已上线
blog.ziyuanzhan.top 技术博客 🚧 规划中
resume.ziyuanzhan.top 在线简历（PDF 版） 🚧 规划中
demo.ziyuanzhan.top 项目演示 🚧 规划中
api.ziyuanzhan.top 后端 API 接口 ⏳ 以后
mail.ziyuanzhan.top 企业邮箱 ⏳ 以后
💡 关键认知：子域名的数量理论上没有上限，实际受 DNS 服务商限制（通常几百到几千个，个人根本用不完）。

三、DNS 记录类型：域名指向的"翻译方式"

在阿里云域名解析后台，你会看到几种记录类型。它们就像不同的"翻译方式"：

记录类型 作用 比喻 我的用法
A 记录 域名 → IP 地址 直接告诉门牌号 以后买服务器时用
CNAME 域名 → 另一个域名 "去隔壁那栋楼" ✅ 现在用：指向 GitHub Pages
MX 邮件服务器 邮局地址 以后配邮箱
TXT 文本验证 贴告示 验证域名所有权
我的实际配置：
记录类型：CNAME
主机记录：@（表示根域名，即 ziyuanzhan.top 本身）<
记录值：binqi.github.io
效果：访问 ziyuanzhan.top → 自动跳转到 GitHub Pages 上的简历页

四、一个域名的完整功能清单

以 ziyuanzhan.top 为例，未来可以扩展的功能：

现在就能做的（免费/低成本）

  • 个人主页：GitHub Pages 托管，0 成本
  • 技术博客：静态博客（Hexo/Hugo），0 成本
  • 项目演示：阿里云 OSS 香港节点，几乎 0 成本
  • 在线简历：HTML 生成 PDF，0 成本

以后能做的（需要服务器）

  • API 服务：Spring Boot 后端，30-50 元/月
  • 文件下载：阿里云 OSS，按量付费
  • 企业邮箱：腾讯企业邮免费版 / 阿里云邮箱
  • 短链接服务：自建，服务器成本
  • 图床：阿里云 OSS，按量付费

五、域名选择的思考

我选 .top 是因为便宜（几十块一年），但有人质疑：".top 能做商业吗？是不是必须 .com？"

答案是：任何域名后缀都能做商业，法律上没有限制。 .com 只是历史惯性带来的"信任感"，不是硬性门槛。

⭐ 实际案例：字节跳动早期用 bytedance.net，拼多多用 pinduoduo.com 但也买了 yangkeduo.com。很多创业公司用 .io、<.co。后缀不重要，产品和运营才重要。

我的策略是：

  • 现在：ziyuanzhan.top 做个人品牌，积累技术影响力
  • 以后开公司：另买 xxx-tech.com 做公司官网
  • 两者并存：个人域名和公司域名分开，互不干扰

六、总结

一个域名 = 一栋大楼，子域名 = 不同的房间。花几十块钱买域名，相当于在互联网上租了一块永久可用的虚拟地产。

对于程序员来说，域名是技术品牌的载体：

  • 求职时，<ziyuanzhan.top 是活的简历
  • 学习时，子域名可以隔离不同项目
  • 创业时，个人品牌和公司品牌可以分离
📌 我的下一步：把 blog.ziyuanzhan.top 和 demo.ziyuanzhan.top 搭起来，让这块"虚拟地产"真正运转起来。
